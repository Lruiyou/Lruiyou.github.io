发表于 2019-08-08 分类于 Java ， 异常处理 本文字数： 4.5k 阅读时长 ≈ 4 分钟

异常处理

拦截可以处理的逻辑错误的异常

  • 在template目录下放error.html错误页面

  • 自定义异常类，并且把错误信息封装成枚举类

    • 定义一个错误信息接口

    • 错误信息的枚举类，并实现错误代码接口

    • 异常类：注意要继承RuntimeException，并且从构造方法传入错误代码接口

    • 定义Handler并使用@ControllerAdvice和@ExceptionHandler注解，@ControllerAdvice只能处理页面级的错误

    • 在出错处加入判断并调用自定义异常，当出错时，便会跳转到error.html页面，并提示用户出错了

处理客户端或服务器出错的异常

  • 自定义Controller并且实现ErrorController,重写getErrorPath()方法。ErrorController可以拦截所有的异常
